**POST /retrain → upload CSV → Execute**
> "Retrain endpoint — uploads new data, trains all three models,
> picks the lowest MSE, saves it. Random Forest wins. No redeployment needed."

## PART 4 — Notebook (75s)
> "The notebook."

**Heatmap**
> "Correlation heatmap — identified the strongest predictors."

**Preprocessing**
> "Label encoding, median imputation, log-transform on skewed features, StandardScaler."

**Loss curves**
> "Loss curves for all three models — train and test — to check for overfitting."

**Results table + scatter plot**
> "Random Forest 0.98, Decision Tree 0.92, Linear Regression 0.84.
> The dataset has missing values, skewed distributions, and non-linear relationships —
> Random Forest handles all of that far better than a linear model.
> The scatter plot shows exactly where linear regression struggles with the spread."

## PART 5 — 4 Questions (2 min)

**Q1 — Loss high or low?**
> "Very low — 0.98. To go further: more recent data, climate features like rainfall,
> or hyperparameter tuning with grid search."

**Q2 — Hyperparameters?**
> "Yes. Random Forest — n_estimators 200, max_depth 10.
> Decision Tree — max_depth, min_samples_split.
> Grid search could optimize these further."

**Q3 — New data?**
> "The retrain endpoint handles it — upload a CSV, all three models retrain,
> best one is saved automatically. No downtime."

**Q4 — CORS?**
> "No wildcard. I whitelisted localhost port 8000 and 10.0.2.2 — the only
> two clients that need access. Explicit methods, headers, and credentials.
> You only allow what you trust."
